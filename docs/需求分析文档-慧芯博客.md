## 1. 系统概述

### 1.1 选题背景

随着互联网技术的飞速发展，博客作为一种重要的信息传播与知识分享平台，在Web2.0时代扮演着不可替代的角色。尽管社交媒体和短视频平台日益盛行，但博客以其深度内容沉淀、专业知识传播、个人品牌塑造等独特优势，依然是技术社区、学术交流、个人创作的重要载体。

近年来，以Medium、Dev.to、Hashnode为代表的现代化博客平台在全球范围内获得了广泛认可。国内如CSDN、掘金、博客园等平台也在持续迭代升级。这些平台的成功表明，优秀的博客系统需要在用户体验、内容管理、社交互动等多个维度做到均衡发展。

本课题"慧芯博客"旨在设计并实现一个功能完善、界面美观、架构合理的现代化博客系统。"慧芯"寓意智慧与初心，期望这个平台能够成为用户记录思考、分享知识、交流思想的精神家园。

### 1.2 选题目的

通过本次课程设计，期望达到以下目的：

1. 综合运用Web全栈开发技术（HTML、CSS、JavaScript、Vue.js、Spring Boot、MyBatis Plus、Spring Cloud等）完成一个完整的Web应用系统
2. 深入理解前后端分离架构的设计理念与工程实践
3. 掌握微服务架构中服务拆分、服务注册与发现、API网关等核心技术
4. 熟练使用Redis作为缓存中间件解决实际业务场景中的性能问题
5. 培养系统设计思维，包括数据库设计、接口设计、架构设计等
6. 提升工程化开发能力，包括Git版本管理、代码规范、注释文档等

### 1.3 系统定位

慧芯博客定位为一款面向技术爱好者与内容创作者的轻量级博客系统，在设计风格上参考X（Twitter）的信息流布局，在功能上涵盖用户管理、文章发布、社交互动、数据统计等核心模块。系统采用B/S架构，支持PC端浏览器访问，具备良好的响应式布局能力。

### 1.4 系统拟解决的实际问题

- 内容创作者的写作发布需求：提供支持Markdown的富文本编辑器，降低写作门槛
- 阅读体验优化：信息流式布局 + 中国风视觉元素，营造沉浸式阅读体验
- 互动交流需求：评论、点赞等基础社交功能
- 内容发现需求：分类、标签、搜索等多维度内容检索
- 数据可视化需求：文章阅读量、用户活跃度等数据统计展示

## 2. 可行性分析

### 2.1 技术可行性

**前端技术栈成熟度：** Vue.js 是目前最流行的前端框架之一，拥有庞大的社区生态和丰富的组件库。Bootstrap 提供了完善的响应式布局方案。HTML5/CSS3/ES6+ 均为成熟的前端基础技术。

**后端技术栈成熟度：** Spring Boot 作为Java生态中最主流的微服务开发框架，已迭代至3.x版本；Spring Cloud 提供了完整的微服务治理方案；MyBatis Plus 极大简化了数据库操作；Redis 作为高性能缓存中间件已被广泛验证。

**数据库技术成熟度：** MySQL 8.0 作为全球最流行的开源关系型数据库，完全满足博客系统的数据存储需求。

**结论：** 所选技术栈均为业界成熟稳定且广泛应用的技术方案，技术可行性高。

### 2.2 经济可行性

本系统为课程设计项目，不涉及商业运营：

- 开发工具：全部使用开源免费软件（IntelliJ IDEA Community Edition、VS Code 等）
- 服务器：本地开发环境，无需云服务器成本
- 数据库：MySQL Community Edition 免费使用
- 第三方服务：Nacos开源版免费、Redis开源版免费

**结论：** 零经济成本，经济可行性高。

### 2.3 操作可行性

- 用户界面：参考X（Twitter）设计风格，信息流式排布，操作直观简洁
- 写作体验：支持Markdown编辑器，所见即所得
- 管理后台：统一的仪表盘管理界面，功能清晰

**结论：** 用户学习成本低，操作可行性高。

## 3. 用户角色分析

系统设计两种用户角色：

### 3.1 普通用户（User）

普通用户是系统的浏览者和互动参与者，主要权限包括：

- 浏览首页文章列表和推荐文章
- 按分类/标签筛选文章
- 搜索文章
- 查看文章详情
- 对文章进行评论和回复
- 对文章进行点赞
- 注册/登录/修改个人信息/上传头像

### 3.2 博主（Blogger）

博主是具有内容发布权限的用户（普通用户可申请成为博主），额外权限包括：

- 所有普通用户权限
- 发布/编辑/删除自己的文章
- 管理自己文章的评论（删除不当评论）
- 查看文章数据统计（阅读量、点赞数、评论数）
- 管理个人主页

## 4. 功能性需求

### 4.1 首页与内容展示模块

**4.1.1 信息流式首页**

- 顶部导航栏：Logo "慧芯博客"、搜索框、分类导航、登录/注册/个人中心入口
- 左侧区域：热门标签云、推荐博主列表
- 中间信息流：文章卡片列表（标题、摘要、封面图、作者信息、发布时间、阅读量、点赞数、评论数），支持无限滚动或分页加载
- 右侧区域：热门文章排行、系统公告

**4.1.2 文章详情页**

- 文章标题、作者信息、发布时间、分类标签
- 文章正文（Markdown渲染）
- 文章底部：点赞按钮、分享按钮
- 评论区：评论列表（支持嵌套回复）、评论输入框
- 右侧悬浮：目录导航（根据文章标题自动生成锚点）

**4.1.3 分类/标签页面**

- 按分类展示文章列表
- 按标签展示文章列表
- 标签云可视化展示

### 4.2 用户系统模块

**4.2.1 注册功能**

- 注册表单：用户名、邮箱、密码、确认密码
- 用户名唯一性验证（前端即时校验 + 后端异步校验）
- 密码强度校验（至少8位，包含字母和数字）
- 邮箱格式校验
- 注册成功自动跳转登录页

**4.2.2 登录功能**

- 登录表单：用户名/邮箱 + 密码
- 支持"记住我"功能（延长Token有效期）
- 登录失败错误提示
- 登录成功跳转首页
- JWT Token认证机制

**4.2.3 个人中心**

- 基本信息展示与修改（用户名、邮箱、个人简介）
- 头像上传与裁剪
- 修改密码（需验证旧密码）
- 我的文章列表（博主）
- 我的评论列表
- 我点赞的文章列表
- 申请成为博主（普通用户可见）

### 4.3 文章管理模块（博主专属）

**4.3.1 发布文章**

- Markdown编辑器（支持实时预览）
- 文章标题输入
- 分类选择（下拉框）
- 标签添加（支持多标签，标签输入建议）
- 封面图上传
- 文章摘要（可自动提取或手动填写）
- 保存草稿 / 发布文章

**4.3.2 文章管理列表**

- 自己的文章列表（支持分页）
- 文章状态标识：已发布 / 草稿
- 快捷操作：编辑、删除、查看
- 每篇文章展示基础数据（阅读量、点赞数、评论数）

**4.3.3 编辑文章**

- 复用发布文章的表单组件
- 预填充原有内容
- 支持更新发布或保存为草稿

### 4.4 评论互动模块

**4.4.1 评论功能**

- 文章底部评论区
- 评论输入框（支持Markdown简化语法）
- 发布评论（需登录）
- 评论列表展示（头像、用户名、时间、内容）
- 支持嵌套回复（二级评论，即回复某条评论）

**4.4.2 评论管理**

- 博主可删除自己文章下的不当评论
- 用户可删除自己的评论

### 4.5 点赞功能

- 文章点赞/取消点赞（toggle模式）
- 点赞数实时更新
- 点赞用户列表展示（可选）

### 4.6 搜索功能

- 全站文章搜索（标题、内容、标签）
- 搜索结果高亮显示关键词
- 搜索结果按相关度排序
- 搜索历史记录（基于Cookie/LocalStorage）

### 4.7 数据统计模块

- 文章阅读量统计（Redis缓存 + 定时同步MySQL）
- 文章点赞数统计
- 用户主页数据概览（总文章数、总阅读量、总点赞数、总评论数）
- 热门文章排行（基于阅读量/点赞数）

## 5. 非功能性需求

### 5.1 性能需求

- 页面首屏加载时间不超过3秒
- 文章列表接口响应时间不超过500ms
- 搜索接口响应时间不超过1秒
- 支持至少100个并发用户同时访问
- Redis缓存热点数据，减少数据库查询压力

### 5.2 安全性需求

- 密码采用BCrypt加密存储，禁止明文存储
- JWT Token设置合理过期时间（访问令牌2小时，刷新令牌7天）
- 接口防SQL注入（MyBatis Plus参数化查询）
- 接口防XSS攻击（前端输出转义 + 后端输入过滤）
- 敏感接口限流保护（网关层实现）
- CORS跨域安全配置

### 5.3 可维护性需求

- 后端代码注释率不低于40%
- 类、方法、字段命名遵循Java命名规范
- 数据库字段名顾名思义，使用下划线命名法
- API接口使用Knife4j生成在线文档
- 模块化设计，各微服务职责单一

### 5.4 可用性需求

- 界面美观，中国风视觉元素
- 响应式布局，适配主流分辨率（1920×1080、1366×768）
- 表单校验提示友好，错误信息明确
- 关键操作（删除文章等）有二次确认

### 5.5 兼容性需求

- 支持Chrome、Edge、Firefox最新版本
- 前端适配PC端为主，兼顾平板设备

### 6.2 博主用例

博主继承普通用户全部用例，额外包括：

- 发布文章
- 编辑自己的文章
- 删除自己的文章
- 管理文章评论（删除不当评论）
- 查看文章数据统计

### 6.3 用例详细描述（节选）

**用例名称：** 发布文章  
**参与者：** 博主  
**前置条件：** 用户已登录且具有博主权限  
**主流程：**
1. 博主点击"写文章"按钮
2. 系统跳转到文章编辑页面
3. 博主输入标题、选择分类、添加标签、上传封面图
4. 博主在Markdown编辑器中编写正文
5. 博主可实时预览渲染效果
6. 博主点击"发布"按钮
7. 系统校验表单数据
8. 系统保存文章，状态设为"已发布"
9. 页面跳转到文章详情页

**异常流程：**
- 6a. 表单校验失败，提示具体错误信息
- 6b. 博主点击"保存草稿"，文章状态设为"草稿"

**用例名称：** 评论文章  
**参与者：** 普通用户/博主  
**前置条件：** 用户已登录，正在查看文章详情  
**主流程：**
1. 用户滚动到评论区
2. 用户在输入框中输入评论内容
3. 用户点击"发表评论"按钮
4. 系统校验评论内容（长度1-500字符）
5. 系统保存评论
6. 评论出现在评论区顶部

**异常流程：**
- 3a. 评论内容为空，提示"请输入评论内容"
- 3b. 评论内容超过500字符，提示"评论不能超过500字"

## 8. 需求总结

慧芯博客系统以"简洁、美观、实用"为核心设计理念，在功能上涵盖了博客系统的完整业务闭环：内容生产（写作发布）→ 内容分发（首页推荐、分类、标签）→ 内容消费（阅读、搜索）→ 用户互动（评论、点赞）→ 数据反馈（阅读量统计）。

在技术架构上采用当前主流的微服务前后端分离方案，前端以Vue.js + Bootstrap构建用户界面，后端以Spring Cloud微服务体系提供稳定可靠的服务支撑，Redis缓存保障高频访问场景的性能表现。

系统设计参考了X（Twitter）的信息流式布局，结合中国风视觉元素，力求在技术实现和用户体验之间取得良好平衡。

## 9. 架构部署补充说明 (v1.1 更新)

### 9.1 解耦开发环境架构设计
在项目开发的中后期，由于远程 Linux 虚拟机的资源瓶颈（尤其当部署完整的微服务矩阵时极易引发 OOM 导致服务器崩溃），本项目实施了**配置解耦与开发环境混合部署**策略：

1. **Nacos 本地化运行**：
   - 将所有微服务（共 7 个）从远端配置中心解绑。移除了 `bootstrap.yml` 及其配套依赖 `spring-cloud-starter-bootstrap`。
   - 这意味着 Nacos 退化为仅提供轻量级的“服务注册与发现”中心，并显式配置为本地运行（`127.0.0.1:8848`），保证项目随时能顺畅连通网络开发而不会再遭遇掉线。

2. **Redis 远程直连**：
   - 考虑到 Redis 属于内存敏感度相对较低的独立服务，为节约开发主机的算力资源，所有微服务的 Redis 环境维持远端 Docker 通信状态（`192.168.238.134:6379`）。
   - 实现方式为在每层的 `application.yml` 里直接注入 `spring.data.redis` 的硬直通节点。

这种本地管理核心服务发现、异地存储非核心中间件的做法，能够在保持微服务架构开发体验的同时，将 OOM 几率降低到了最小。
